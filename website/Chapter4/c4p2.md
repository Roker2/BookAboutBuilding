# Параграф 2: Утилиты grep, strings, sed, shim, patchelf

Данный параграф описывает какими утилитами можно проводить поиск информации и как патчить файлы, в том числе проприетарные библиотеки.

### grep

Ваш лучший друг в поисках ключевых слов буквально везде: от текстовых файлов до блобов.

Обычно я использую флаги `-InRI` при поиске в текстовых файлах, к примеру, в конфигах или исходном коде. Пример:

```bash
grep -InRI "DEVICE_PACKAGE_OVERLAYS"
```

Пример вывода:

```
device.mk:29: DEVICE_PACKAGE_OVERLAYS += $(LOCAL_PATH)/overlay
device.mk:30: DEVICE_PACKAGE_OVERLAYS += $(LOCAL_PATH)/overlay-lineage
```

Если мне требуется уже поискать в проприетарных файлах что-то, то использую флаги `-ril`. Пример:

```bash
grep -ril "persist.camera.debug.logfile"
```

Пример вывода:

```
vendor/lib/libmmcamera_hdr_gb_lib.so
vendor/lib/libmpbase.so
vendor/lib/liboptizoom.so
vendor/lib/libseemore.so
vendor/lib/libtrueportrait.so
vendor/lib/libubifocus.so
vendor/lib/libchromaflash.so
```

### sed

А это - лучший друг для патчинга библиотек и конфигов! К примеру, можно заменить один текст на другой прямо в блобе. Пример замены проперти на другой:

```bash
sed -i "s|persist.camera.debug.logfile|persist.vendor.camera.dbglog|g" libmmcamera_dbg.so
```

Отмечу, что текст можно сделать меньше, добавив вместо символа `\x00`. Пример:

```bash
sed -i 's|android.hidl.base@1.0.so|libhidlbase.so\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00|g' libname.so
```

Но заменяемый текст не должен быть больше, чем исходный текст. Таким образом можно заменить и ASM команды в виде байтов:

```bash
sed -i 's|\x12\x34\x56\x78|\x87\x65\x43\x21|g' "libname.so"
```

### shim

Shim - самописная библиотека, которая восполняет отсутствующие функции или заменяет их. Функции делают пустыми или вручную реализуют логику.

Shim на [камеру](https://github.com/zeelog/android_device_xiaomi_mido/blob/lineage-21.0/libshims/libshims.cpp) для mido. 

```cpp
#include <stdint.h>

namespace android {
    // libandroid.so
    extern "C" void ALooper_forThread() {}
    extern "C" void ALooper_pollOnce() {}
    extern "C" void ALooper_prepare() {}
    extern "C" void ALooper_wake() {}
    extern "C" void ASensor_getMaxRange() {}
    extern "C" void ASensor_getMinDelay() {}
    extern "C" void ASensor_getName() {}
    extern "C" void ASensor_getVendor() {}
    extern "C" void ASensorEventQueue_disableSensor() {}
    extern "C" void ASensorEventQueue_enableSensor() {}
    extern "C" void ASensorEventQueue_getEvents() {}
    extern "C" void ASensorEventQueue_setEventRate() {}
    extern "C" void ASensorManager_createEventQueue() {}
    extern "C" void ASensorManager_destroyEventQueue() {}
    extern "C" void ASensorManager_getDefaultSensor() {}
    extern "C" void ASensorManager_getInstanceForPackage() {}
    // libandroid_runtime.so
    extern "C" void _ZNK7android6Sensor11getMaxValueEv() {}
}
```

Тут видно, что функции пустые. И название функции `_ZNK7android6Sensor11getMaxValueEv` выглядит странно. Оно такое, так как название функции в библиотеке меняется после компиляции.

### patchelf

Данная утилита используется для замены, добавления и удаления зависимостей в блобах. Используется это для того, чтобы можно было обновить библиотеки и исполняемые файлы под новые зависимости, прописать шимы или новые библиотеки, а также удалить уже несуществующие зависимости.

Пример добавления зависимости (шима):

```bash
patchelf --add-needed "fakelogprint.so" vendor/lib64/hw/fingerprint.goodix.so
```

Пример удаления зависимости:

```bash
patchelf --remove-needed "libbacktrace.so" vendor/bin/gx_fpd
```

Пример замены зависимости:

```bash
patchelf --replace-needed "libstagefright_foundation.so" "libstagefright_foundation-v33.so" vendor/lib64/libdlbdsservice.so"
```
